raw

提供报表的抽象模型
thanks to https://raw.githubusercontent.com/densitydesign/raw/master/lib/raw.js
& d3.js

方法

getChartTypes #

根据 raw.models 获取已定义 chart 类型

raw.chart #

创建一个图表

方法参数:

参数名 类型 描述 必选 支持版本
id string 图表 id

返回值: chart

chart.draw #

图表的绘制方法

方法参数:

参数名 类型 描述 必选 支持版本
canvas dom 画布节点
data array 格式化后的数据
config object 配置

返回值: chart.category

chart.model #

获取/设置图表 model id，比如 chart 'PIE' && 'VUE.PIE' 实际可以基于同一个 model

方法参数:

参数名 类型 描述 必选 支持版本
string

raw.model #

创建一个空的模型

返回值: 模型

model.map #

定义模型的 map 方法

方法参数:

参数名 类型 描述 必选 支持版本
数据格式化函数 function

返回值: model

model.dimension #

定义模型的维度

方法参数:

参数名 类型 描述 必选 支持版本
维度的唯一标识 string

返回值: 该维度

model.dimensions #

获取模型的所有已定义维度

返回值: 维度

model.isValid #

校验模型各维度赋值是否满足 required 限制

返回值: bool

model.clear #

清空模型各维度

返回值: undefined

dimension #

已定义的维度，用以接收获取数据

方法参数:

参数名 类型 描述 必选 支持版本
object array|object 一行数据记录
ext any 扩展参数，作为第二个参数传递给 accessor，用以做猥琐的事情

返回值: 参数缺省:

dimension.info #

获取该维度的所有信息

返回值: object

dimension.format #

设置/读取维度共用的格式配置参数

方法参数:

参数名 类型 描述 必选 支持版本
格式化参数 any

返回值: 参数缺省返回

dimension.accessor #

读取/设置数据的获取器，默认 accessor 直接返回值，接受参数 (value, ext, index)

方法参数:

参数名 类型 描述 必选 支持版本
function

dimension.multiple #

读取/设置维度值的是否多个条数的配置

方法参数:

参数名 类型 描述 必选 支持版本
bool

dimension.required #

设置/读取维度值条数限制

方法参数:

参数名 类型 描述 必选 支持版本
number|array

返回值: 视参数是否缺省

dimension.types #

设置/获取维度值的类型

方法参数:

参数名 类型 描述 必选 支持版本
'METRIC', any 'DIMENSION' 指标或维度

返回值: 视参数是否缺省

dimension.type #

设置/获取维度第一个值的类型

dimension.clear #

清空维度值

dimension.add #

向维度内添加一个值，会根据 required, multiple 配置自动限制值的条数
